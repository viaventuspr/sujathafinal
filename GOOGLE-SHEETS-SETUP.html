2
Enable Google Sheets API
3 min
  1. Go to console.cloud.google.com
  2. Create a new project (or use existing) — name it "Sujatha Book Orders"
  3. Click APIs & Services → Library
  4. Search for "Google Sheets API" and click Enable
ℹ️ Google Cloud has a generous free tier — Google Sheets API calls are free for normal usage. You won't be charged for book orders.
3
Create Service Account & Download Key
5 min
  1. In Google Cloud Console → APIs & Services → Credentials
  2. Click + Create Credentials → Service Account
  3. Name: "book-orders-bot" → click Create and Continue
  4. Role: select Editor → click Done
  5. Click on the service account email you just created
  6. Go to Keys tab → Add Key → Create new key → JSON
  7. A .json file will download — keep it safe!
Convert JSON to Base64 (for Netlify)

Run this command in your terminal (Mac/Linux) where the JSON file is downloaded:

base64 -i your-service-account-file.json | tr -d '\n'

On Windows (PowerShell):

[Convert]::ToBase64String([IO.File]::ReadAllBytes("your-service-account-file.json"))

Copy the entire base64 output — this is your GOOGLE_SERVICE_ACCOUNT_KEY.

Service Account Email

Inside the JSON file, find client_email — it looks like:

book-orders-bot@sujatha-book-orders.iam.gserviceaccount.com

Save this email — you'll need it for Step 4.

⚠️ Security: Never share or commit the JSON key file. The base64 version goes into Netlify's secure environment variables — never in your code.
4
Share the Sheet with Service Account
1 min
  1. Open your Google Sheet
  2. Click Share (top right)
  3. Paste the service account email:
    book-orders-bot@your-project.iam.gserviceaccount.com
  4. Set permission to Editor
  5. Click Send (uncheck "Notify people")
✅ The service account now has permission to append rows to your sheet automatically when orders are placed.
5
Add Environment Variables to Netlify
2 min
  1. Go to app.netlify.com → your site
  2. Click Site configuration → Environment variables
  3. Add these two variables:
GOOGLE_SERVICE_ACCOUNT_KEY
Paste the entire base64 string from Step 3 here. It will be very long — that's correct.
GOOGLE_SHEET_ID
The Sheet ID from your Google Sheets URL (the long string between /d/ and /edit).
  1. Click Save
  2. Go to Deploys → Trigger deploy → Deploy site (to restart with new env vars)
🔐 Netlify env vars are encrypted and never exposed to visitors. Your service account key is completely safe.

Troubleshooting
Orders not appearing in sheet
  1. Check that GOOGLE_SHEET_ID is correct (no extra spaces)
  2. Confirm the sheet tab is named exactly Orders
  3. Verify the service account email was shared with Editor access
  4. Check Netlify Function logs: Site → Functions → order → logs
403 Forbidden error in logs

The service account doesn't have access to the sheet. Re-share the sheet with the service account email as Editor.

Orders work but sheet column is wrong

Make sure the sheet tab name is exactly Orders (capital O, no spaces). The function writes to Orders!A:K.

ℹ️ Fallback: Even if Google Sheets fails, customers still see the order confirmation and get a WhatsApp confirmation button. No order is ever lost.
